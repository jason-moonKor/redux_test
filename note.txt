## Rudux 개요

리덕스는 단일 스토어이다

단일 스토어 사용준비하기
- import redux
- 액션을 정의한다
- 액션을 사용하는, 리듀서를 만든다
- 리듀서들을 합친다
- 최종 합쳐진 리듀서를 인자로, 단일 스토어를 만든다

준비한 스토어를 리액트 컴포넌트에서 사용준비하기
- import react-redux
- connect 함수를 이용해서 컴포넌트에 연결

### 리덕스 액션

- 액션을 객체이다.

- 두가치 형태이 액션
-- {type: "TEST"} //payload가 없는 액션
-- {type: "TEST", params: "hi" } //payload가 있는 액션을

- type은 필수 프로퍼티이고, type은 문자열 이다

### 액션 생성자

- 액션을 생성하는 함수를 "액션 생성자"라고 부른다
- 함수를 통해 액션을 생성, 액션 객체를 리턴한다

#### 리덕스의 액션은 어떤 역할?

- 액션 생성자를 통해 액션을 만들어 낸다
- 만들어낸 액션 객체를 리덕스 스토어에 보낸다
- 리덕스 스토어가 액션 객체를 받으면 스토어의 상태 값이 변경된다
- 변경된 상태 값에 의해 상태를 이용하고 있는 컴포넌트가 변경된다
- 액션은 스토어에 보내는 일종의 인풋이라 생각할수있다.


### 리덕스 리듀서

- 액션을 주면 액션이 적용되어 결과를 만들어 준다
- 함수이다
- 리듀서를 통해 스테이트가 달라졌음을 리덕스가 인지한다



## React-redux

- Provider 컴포넌트를 제공해준다
- connect 함수를 통해 "컨테이너"를 만들어준다
- 컨테이너는 스토어의 state와 dispatch(액션)을 연결한 컴포넌트에
  props로 넣어주는 역할을 한다
- 필요한것 :
-- 어떤 state, dispatch를 어떤 props에 연결할 것인지에 대한 정의
-- 그 props를 보낼 컴포넌트를 정의
